# Individual vs Global fit — validation demo

Validates the per-residue individual-vs-global comparison and the jackknife
global-fit validation against three synthetic datasets with **known** ground
truth. Each has 5 residues at 2 fields, generated by the exact Matrix model
with a strong minor state (`pB=0.1`, 1% noise).

| Scenario | Ground-truth kex per residue | Expectation |
|----------|------------------------------|-------------|
| `global` | all `1000` | whole-dataset verdict **global**, jackknife robust |
| `individual` | `400, 700, 1000, 1500, 2500` | whole-dataset verdict **individual**, jackknife unstable |
| `mixed` | `1000, 1000, 1000, 1000, 2500` | lone outlier **R5 flagged**; **global restored** once R5 is removed |

## Run

```bash
bash demo/run_demo.sh          # generate -> fit -> evaluate
# or with a specific interpreter:
PYTHON=/path/to/python bash demo/run_demo.sh
```

`make_scenarios.py` writes each scenario's `.dat`, `.conf`, and `truth.json`
into `demo/<scenario>/`; `evaluate.py` parses the `##### model_comparison` /
`##### jackknife` JSON blocks from the captured `sk_run` stdout and checks the
verdict against ground truth (exit 0 = all pass).

## What the mixed case shows

Least-squares global fitting is **not robust**: one high-leverage outlier
(R5, `kex=2500`) dominates the total chi² and drags the shared `kex` away from
the true consensus (`1000 -> ~1126`), so *every* residue then prefers its own
fit. This is a correct diagnosis that the single-`kex` model is inappropriate,
not a tool failure. The tool pinpoints the culprit — R5's `z(kex)` and
`chi2_global` dwarf the others (~3x and ~16x) — and removing R5 restores a
clean global fit (jackknife relative SE drops from ~13% to <1%, `kex` recovers
to ~995). Note the jackknife's per-residue `|delta_kex| > 2 SE` flag is *masked*
by the outlier inflating the SE (single-deletion breakdown point = 1); the
per-residue `z(kex)` / `chi2_global` diagnostics are the sensitive signal.

Generated `.dat` / `.conf` / `out*.txt` / `truth.json` / `.log` / `.pdf` are
reproducible and git-ignored.
